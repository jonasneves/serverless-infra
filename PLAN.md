# Roadmap

## Current

- Container-based (image or Dockerfile)
- Quick Tunnels (zero-config, random URLs)
- Managed Tunnels (stable URLs, custom domains)
- Keep-alive monitoring (restart crashed containers)
- Multi-arch support (AMD64, ARM64)

## Next

### Starter Templates

Workflow templates for GitHub's Actions tab with common Dockerfile patterns.

### CLI

`npx create-preview-workflow` to scaffold workflow + Dockerfile.

### Multi-Container

Support docker-compose for multi-service deployments.
